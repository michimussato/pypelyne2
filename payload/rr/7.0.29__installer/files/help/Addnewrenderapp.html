Render



To render with new application, you need to add these files:


  • Render Config File            [RR]\render_apps\_config
  • Install Path                      [RR]\render_apps\_install_paths

   (Only required if you use the <Exe> variable in your render config, otherwise hardcode the path.

   <Exe> requires to add the render application to the client config)


Submit


To automatically fill out the basic scene settings in the rrSubmitter you have two options:


A) Create a plugin inside your render application.  

Examples: Softimage, Maya, 3dsMax.

Required files:


  • Submission Plugin                [RR]\render_apps\_submitplugins\
  • Install Path                        [RR]\render_apps\_install_paths

   (rrWorkstationInstaller automatically installs the plugin into all versions)


B) Create a Scene Parser (SDK link) plugin.

The submitter reads the scene file.

Examples: Nuke, Shake, Fusion.

A Scene Parser is required if you want to load the scene file into the rrSubmitter.

Alternatively, you can also create two text files to let the rrSubmitter know what to do with that file.

As an example please see the files [RR]\plugins\submitter_sceneparser\3D05__MakeVideo.desc.*





Environment


Most of the render applications require to set some environment variables.

May it be to set the license server, the path to preferences or plugins.

Therefore you need Setenv Files.

Move RR folder

Installation ›› Additional Info ››


This descriptions shows you how you can move the whole RR Root folder.

The network folder that includes all RR application files and config files.

This is NOT a description if you want to execute the rrServer on a different machine.
This description requires that your old RR installation runs with all clients connected.


  1. RR Program Files:
    If you want to keep your configuration, then copy all files from the old directory to the new directory.
    Otherwise, do a clean install into the new folder.

  2. Startup scripts:
    The startup scripts are all win_*.bat (windows) and all ls_*.sh (linux) files in the base of the RR folder.
    If you have done a clean install:
           Copy all startup script files from the new installation to the old one.
    If you have copied the old folder:
           You have to edit all startup script files in the existing RR root folder with any text editor.
           Change the path after RR_ROOT to the new path.
           Inside the ..rrWorkstation_installer file, change the path after RR_ROOT_INSTALLER. This one is the most important startup script.

  3. Access rights:
    Check if the access rights are right. See File Access Rights.

  4. rrServer:
    Reinstall the rrServer Service via rrWorkstationInstaller.

  5. rrWorkstationInstaller step I:   Setup service installer:
    Start the new rrWorkstationInstaller on one client and reinstall it.

  6. rrWorkstationInstaller step II:    Distribute to all clients:
    Start rrControl and create a new command job via the menu "clients".
    Choose "Reinstall ALL (...)"
    Enable the job.




Relink old RR folder:

You can install a config relink in case there are machines, that you cannot update right now with the rrWorkstationInstaller.

Perhaps they are offline at the moment.

These machines will still start the executables from the old RR share, but load the config of the new one.

Inside the old RR share, create the files

[RR]\sub\cfg_global\relink_win.cfg

[RR]\sub\cfg_global\relink_lx.cfg

[RR]\sub\cfg_global\relink_mac.cfg


Inside these files, write one line Root= which points to the new location, e.g.

Root=\\redhat\renderfarm\rr6\


Note: if you have created the batch file to re-install the service, then they will catch the job once they get online.

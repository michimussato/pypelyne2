Batch


via commandline



Please check the following files.

They all execute Maya commands in different ways.


RR\render_apps\_config\additional\3D02__Maya2008__FluidCache.cfg

It uses the Layer name to specify the fluid object in your maya scene and the image name is the fluid file name generated.

RR\render_apps\_config\3D02__Maya2008__CreateRenderMi.cfg

loads a hardcoded mel script and executes the function in that file.

RR\render_apps\_config\additional\3D02__Maya2009__MelOnce_Melfile.cfg

loads a mel script that you enter as image name of the job. And executes the function in that file.

RR\render_apps\_config\3D02__Maya2009__MelOnce.cfg

executes the commands you enter as "user info" for the job.

via render desc file


Please check the file

RR\render_apps\_config\3D02__Maya2012__CreateRenderVray.cfg


It uses a new render description file for maya as "renderer"

-r vrayexport


The render description file is located in

RR\render_apps\renderer_plugins\maya\win_x64\2013\rendererDesc\



Arnold/MtoA


Farm-wide Install


Royal Render changes the environment that you do not have to install MtoA on all machines.



Note: This description is for Maya 2013 x64 Windows.

Change the version according to your installation. So if you use Maya 2014, then you should use "2014" instead of "2013". If you use linux, use lx instead of win


  • Please copy your MtoA files into:

     [RR]\render_apps\renderer_exe\maya\win_x64\mtoadeploy_2013

  • Please copy your MtoA/Arnold shaders into:

     [RR]\render_apps\renderer_exe\maya\win_x64\mtoadeploy_2013\shaders

  • Please edit the file [RR]\render_apps\renderer_exe\maya\win_x64\mtoadeploy_2013\mtoa.mod.

     Update the path to match this path.

     The line should look something like this:

     + mtoa * \\smaug\Renderfarm\_RR6\render_apps\renderer_exe\maya\win_x64\mtoadeploy_2013


If Maya finds the plugin, but states that it cannot load the module, please install the Visual Studio 2012 Redistributable Package on the client:

http://www.microsoft.com/en-us/download/details.aspx?id=30679



Multiple MtoA versions


It is possible to use multiple MtoA versions for one Maya version.

There are two ways:

  1. Edit the file [RR]\render_apps\_setenv\win\maya.bat and change the part
    "RR_MTOA_BASE=%rrSharedExeDir%mtoadeploy_%rrExeVersionMajor%"
    to
    "RR_MTOA_BASE=%rrSharedExeDir%mtoadeploy_%rrExeVersionMajor%_%rrJobRendererVersion%"

    Send a job, check the render log and search for "MtoA installation path is set to".
    This is now the folder you have to use.


  2. Use the rrEnvline or rrEnvfile for a job to set the variable RR_MTOA_BASE.
    To verify it, send a job, check the render log and search for "MtoA installation path is set to"





Ass N Render


You can change the renderer to "arnold- Ass N Render" at the rrSubmitter.

Instead of starting Maya and rendering the Maya scene with MtoA, it will  start Maya, export .ass files to a local temp folder and renders these .ass files with kick.

This will save all the memory that would be required by Maya for opening the scene file.


Note:

You must not set the render options in Maya to"Export Ass" or "Export Ass and Kick".

Set the Arnold render settings to "Interactive".


Note:

KSO mode is not supported.



Export .ass


To export .ass files, change the Arnold render settings in your Maya scene. System/ Render Settings/ Render Type to "Export Ass".

The rrSubmit plugin read the .ass export filename instead of the usual image filename.

And it changes the renderer to arnold-exportAss.

Fluid Caches


There is a render config file for rendering Maya fluids.

To use it, you have to copy the file

RR\render_apps\_config\additional\3D02__Maya2008__FluidCache.cfg

into RR\render_apps\_config\3D02__Maya2008__FluidCache.cfg


In the file is a description how to submit it via console.

Or you can change the renderer in rrSubmitter and change the values as shown in the render config:

# Variables you have to set differently from a default rendering:

# renderer= "FluidCache"

# layer: name of the fluid shape

# imageDir: Folder for the cache files

# imageFilename: Filename of the cache file. IMPORTANT: The filename has to end with "Frame".

# ImageExtension: ".mc"

Error - Fluids


If you render with fluids, you might encounter a massive slowdown.

Even if you have cached the fluid files.

VRay will fix this for the upcoming version, but for now the solution is to

enter the following into the "pre render Mel" script in Maya to get it to behave:

$i = `getAttr "defaultRenderGlobals.startFrame"`; currentTime -edit $i; print`currentTime -q`;



Error - Render Element Separator


There seems to be a bug in VRay. (at least version 1.5 Sp4)

No matter what you use as Render Element Separator ( . _ - ), VRay always renders with a . (point).


Therefore RR cannot find the images.

Please use the default setting . (point) as Render Element Separator.

Two VRay versions with one Maya version


Using two different VRay versions with one Maya version gonna be tricky.

As VRay has to have files inside the Maya app folder. It is not a default plugin that works with environment variable paths only.

But is possible if you use do not render with Maya on multiple client threads and the client is not used by any artist. (Maya must not be running, otherwise the dll is in use and cannot be replaced)

And you have to to a farm installation of VRay, see Help section Maya/VRay


There are two ways:

  1. Edit the file [RR]\render_apps\_setenv\win\maya.bat and change the part
    "RR_VRAY_BASE=%rrSharedExeDir%vray%rrExeVersionMajor%"
    to
    "RR_VRAY_BASE=%rrSharedExeDir%vray%rrExeVersionMajor%_%rrJobRendererVersion%"

    Send a job, check the render log and search for "VRAY installation path is set to".
    This is now the folder you have to use for your farm-wide installation




Redshift

Farm-Wide installation:

  • Install Redshift on your workstation.
  • Copy the folder c:\ProgramData\Redshift\     to    [RR]\render_apps\renderer_exe\maya\win_x64\
  • Rename this folder: Add an underscore and the redshift version to the folders name. E.g.:  Redshift_1.0.64
    The resulting directory is then for example [RR]\render_apps\renderer_exe\maya\win_x64\Redshift_1.0.64



Hint:

If you are unsure about the right folder name, then submit a redshift job and let it render.

Open the render log file of the job and search for  "RedShift Maya plugin installation path is set to"

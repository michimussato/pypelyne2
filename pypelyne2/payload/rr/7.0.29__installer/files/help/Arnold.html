Please see the Softimage help section.


Setup


By default Arnold is configured to use a Shared Executable.



Important note SitoA/ MtoA/ HtoA:

This description is for Arnold Standalone, not SitoA/ MtoA/ HtoA.

If you have SitoA/MtoA/HtoA, then it is possible to build your Arnold Standalone folder by taking files from an SitoA/MtoA/HtoA folder and placing them into the /bin  subfolder of the Arnold folder.

Just make sure the kick executable is in the right folder in the end.



To setup Arnold, please follow these steps.

Note: this description is for Arnold Standalone 4.0.6.0 and windows x64, please modify regarding your OS and version

  • Copy your Arnold Standalone folder to

     [RR]\render_apps\renderer_exe\arnold\win_x64\Arnold-4.0.6.0\

     You probably should only copy the \bin folder to keep the file count low.

     The resulting kick.exe will be located in [RR]\render_apps\renderer_exe\arnold\win_x64\Arnold-4.0.6.0\bin\kick.exe


  • Optional:
    Edit [RR]\render_apps\_setenv\win\arnold.bat, remove the comment ('rem') and set your license server name.




SitoA/MtoA/HtoA:

If your .ass files have been exported by SitoA/MtoA/HtoA, then you need to copy the shader dlls from your SitoA/MtoA/HtoA installation into Arnold Standalone.



For example SitoA:

Copy from the folder        [YourWorkgroup]\Addons\SItoA\Application\Plugins\bin\nt-x86-64\

the files

sitoa.dll

sitoa_curves_proc.dll

sitoa_shaders.dll

into the folder

[RR]\render_apps\renderer_exe\arnold\win_x64\Arnold-4.0.6.0\bin\






Submission


Just like every other archive renderer type, you have to load one file of your .ifd sequence into the rrSubmitter.

The rrSubmitter recognizes the frame range of your sequence and output name.

You can also start the rrSubmitter with the file as commandline parameter.


All Archive/Standalone renderer are based on a scene file per frame in Royal Render.

You can not export a complete animation into one file, you need one file per Frame.

name.#.ext



Infos


Tiles EXR


Arnold writes Tiled EXR.

Scanline EXR are faster to load in some comp apps.

Please use the CropEXR of RR to convert them into scanline EXR files.

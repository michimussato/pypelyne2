Auto-Mount


You need to auto-mount the RR share at startup.

The same for all your mounts to your project data.


Important:

Most auto-mount tutorials tell you to add the mount into System Preferences/Accounts/Login Items.

But these are only executed if you login with that user.

And the files are only accessible by that user.

So if your daemon uses a different user name, it can NOT access the date!



1) NFS shares:

If your filserver can be accessed via an NFS share, then use the application Utilities/Directory Utility to mount the share.



2) Samba shares:

Apple does not like samba and therefore does not support auto-mounting of samba Shares well.

All solutions so far are not very good.

This is a collection of "solutions" so far:


2a) Samba, Mount via /etc/fstab:

If you are using a samba share (windows), then follow these steps:

Please create a file named /etc/fstab and add a line for each mount.

A line should look like this:

YourFileServer:/YourShare /Volumes/YourShare url automounted,url==cifs://SmbUserName:SmbPasswd@YourFileServer/YourShare 0 0

(You have to replace YourFileServer, YourShare, SmbUserName, SmbPasswd.)

Note: I have testes this way and I still have the issue that the file permissions are not set for all users.


2b) Samba, Mount via auto.smb:

http://www.stress-free.co.nz/automounting_samba_shares_in_leopard


2c) Samba, Mount via script:

The mount_samba command has an option for file and directory mode. Probably writing a shell script with that command is the best alternative.


2d) Samba, NFS from windows:

So far the best solution is to install for example "Windows Services for UNIX" on the windows machine to create NFS shares.

Then mount them via the application Utilities/Directory Utility.

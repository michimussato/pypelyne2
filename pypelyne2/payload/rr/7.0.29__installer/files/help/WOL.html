WOL does not work

You right-click on an offline client in rrControl and send a WOL command.

But the machine does not turn on. (check the machine itself, perhaps WOL works, but it is not able to start the client).


TCP subnet mask:

If your TCP subnet mask is not 255.255.255.0 , then you have to change it in rrConfig, tab Clients global (enable "show advanced settings").



Some things to check:

  • Check the next chapter with the arp -a command to re-check for the right MAC addresses to use.
  • RR settings - Client MAC address
    If you have multiple network cards in the machine, then it can be that the wrong one is entered in the client config.
    On windows, open console with start/cmd.exe.
    Enter ipconfig /all
    It shows you the "Physical Address" of all adapters.
  • Network card driver - update
    Install latest drivers for network card (not using Windows check, manually searching and installing from manufacturers driver downloads)
  • Network card driver - settings
    Check if WOL is enabled in the hardware device manager of the client.
    Windows: Open the Device Manager, choose the right network card, tab "Advanced".
    Search for WOL or Wake Up. It has to be set to "Magic packet" or "Both".
    If there is a Wake from Shutdown, enable that setting.
  • Windows Power Management setting
    Open the Device Manager, choose the right network card, tab "Power Management".
    Allow this device to wake the computer via magic packets.
  • BIOS
    Note: Depending on your BIOS, you might have one or multiple of these settings
    • Power Management section: disable Deep Sleep.
    • Power Tab, APM configuration screen, Power On By PCIE Devices needs to be enabled.
    • Advanced tab, Chipset screen, South Bridge screen, enable  "Wake on LAN from S5" (AMI/ American Megatrends BIOS)
    • Power  Management Screen, Enable WOL from Soft Off (Biostar BIOS)
    • Power screen, Hardware Power Management screen,  enable "S5 Wake on LAN" (HP Bios)

  • RR settings - global WOL settings
    Open rrConfig, tab "Clients (global)", enable "Show advanced settings" at the bottom.
    Check if the subnet mask matches the settings in your network configuration of the system.
    You can try to change the WOL port to 80. (port 80 is used for the http:// internet. It might be that a device in your company blocks every other port for security reasons)



Windows:  Test WOL command transmission


You can test if the WOL command send by RR is at least transmitted to your machine.


  1. Download this application:
    http://www.depicus.com/wake-on-lan/wake-on-lan-monitor.aspx
  2. Execute it on the machine with the WOL problem (no need to install)
  3. By default, RR uses the port 2304 for WOL. Enter this into UDP port.
  4. Send a WOL command to the machine with the WOL problem.
  5. If the monitor app catches the WOL command, then everything is right within your network.
    So the issue is the computer (BIOS, network card, settings)
    If nothing is received, then the issue is either a firewall at your rrServer, a network switch or network router.
